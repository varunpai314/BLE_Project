## The Problem
The Chief Quality Assurance Officer at **Wenlock District Hospital, Mangaluru** wants to automatically track inpatient locations — when patients are taken to MRI, X-Ray, Operation Theater, etc. Nurses are too busy to update locations manually.

## The Solution
A **BLE-based indoor patient zone detection system** using:
- ESP32 beacon on patient wristband
- Android phones (or ESP32s) at zone entrances as gateways
- Dart server on a laptop for zone logic
- Flutter Web dashboard for live tracking

## Core Architecture
```
[ESP32 Beacon - wristband]
        ↓ BLE advertisement
[Android Gateway App - Flutter]
        ↓ HTTP POST over local WiFi
[Dart Server - headless, shelf]  ←→  [Flutter Web Dashboard - browser]
```

## Zone Logic
- **Zone = strongest RSSI gateway** for that beacon in the last 10 seconds
- No triangulation — simpler and reliable enough for zone-level accuracy

## Key Decisions Made
1. Zone-based proximity (not triangulation)
2. Server-side zone logic — gateways just forward raw RSSI
3. Local WiFi only — no internet dependency (see v1.1 for cloud layer)
4. Pure Dart server — avoids cross-compile issues
5. Flutter Web dashboard — opens in any browser on local network
