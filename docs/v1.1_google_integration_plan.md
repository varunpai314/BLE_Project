## Technologies to Add

### 1. Firebase Firestore
**Why:** Replace in-memory `_patients`/`_events` maps. Data survives server restarts. Historical data is required for AI anomaly detection.  
**How:** Dart server writes to Firestore on every zone change. Dashboard reads from Firestore as fallback.  
**Offline-first:** Core BLE + WiFi system works 100% without internet. Firestore is a sync layer — if offline, data stays local and syncs when connection returns.

### 2. Firebase Cloud Messaging (FCM)
**Why:** Push alerts to nurses' phones without them having the dashboard open.  
**How:** Dart server sends FCM push via HTTP when anomaly is detected. Nurses receive notification on their Android phones.  
**Target device:** Separate nurse phone (not the gateway phone). Nurse also has access to web dashboard.

### 3. Firebase Authentication
**Why:** Secure login for nurses and the CQA admin officer. Different roles = different views.  
**How:** Firebase Auth (email/password). Dashboard checks auth state before showing patient data.  
**Future:** Role-based alerts — nurse gets alerts only for their ward's patients.

### 4. Firebase Hosting
**Why:** Host the Flutter Web dashboard publicly (on local network AND optionally internet).  
**How:** `flutter build web` → `firebase deploy`. URL is shareable with any hospital staff.  
**Bonus:** Another Google technology checked off for Solution Challenge.

### 5. Gemini API (AI Core)
**Model:** Gemini 1.5 Flash (free tier: 15 RPM, 1M tokens/day — sufficient for demo + hackathon)  
**Accessed via:** REST API from Dart server, no extra SDK needed.

#### AI Feature A — Anomaly Alerts ("Smart Watch") 🚨
> *"Patient P-3 has been in the Operation Theater for 4h 12m. Standard procedure time is 2-3 hours. Alert dispatched to Ward Nurse via FCM."*

- Time-in-zone thresholds configured per zone (OT: 3h, MRI: 45min, etc.)
- Gemini generates the human-readable alert message in plain English
- Delivered via FCM to the assigned nurse's phone
- **Clinical value:** Nurses catch delayed discharges, procedure complications, forgotten transfers

#### AI Feature B — Natural Language Query ("AI Assistant") 💬
> Nurse types: *"Which patients have been waiting the longest?"*
> Dashboard replies: *"Patient P-2 (OPD Waiting, 1h 23m), Patient P-1 (General Ward, 47min)"*

- Gemini receives current `_patients` JSON as context
- Answers in plain language
- No fine-tuning needed — pure prompt engineering with Gemini Flash
- Future: Kannada language support

#### AI Feature C — End-of-Shift Summary Report 📋
> One-click generation. Gemini receives full day's event log and produces:
> *"12 patients tracked today. Avg OPD wait: 23 minutes. 2 anomalies flagged. Longest stay: Patient P-1, General Ward, 6h 42m."*

- CQA officer gets a ready report, no manual compilation
- Demonstrates tangible value to hospital administration

## Updated Architecture (with Firebase + Gemini)
```
[ESP32 Beacon] → BLE → [Android Gateway] → HTTP POST → [Dart Server]
                                                              │
                              ┌───────────────────────────────┤
                              │                               │
                    [Local in-memory state]          [Firestore sync]
                       (real-time, offline-safe)      (persistence, history)
                                                              │
                                              [Gemini API] ← patient context
                                                    │
                                             [FCM Push alerts]
                                                    │
                                         [Nurse Android Phone]
                                                    │
                                      [Flutter Web Dashboard]
                                      (Firebase Hosting, Auth)
```

## Implementation Order
1. Firebase Firestore — persistence layer first
2. Gemini anomaly detection — biggest clinical demo value
3. FCM push notifications — ties it together
4. NL query assistant on dashboard — impressive live demo
5. Firebase Auth — polish before submission

## Offline-First Guarantee
- BLE scanning: ✅ always local
- HTTP POST to Dart server: ✅ local WiFi only
- Zone detection logic: ✅ in-memory, no internet needed
- Firebase sync: ⚡ optional, graceful failure if no internet
- Gemini alerts: ⚡ optional, falls back to rule-based local alert
- FCM: ⚡ requires internet, fallback = dashboard notification

## Constraints
- Gemini free tier: 15 req/min — for anomaly checks every 60s per patient, this is fine
- Firestore free tier: 50k reads/day, 20k writes/day — more than enough for demo
- FCM: free forever for basic push
