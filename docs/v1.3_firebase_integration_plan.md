## What Firestore Adds
- Zone change events persist after server restart
- Historical data available for Gemini AI analysis
- Dashboard can show data even if Dart server is briefly offline
- Foundation for future multi-hospital support

## Offline-First Guarantee (unchanged)
- BLE scanning → always local
- HTTP POST to Dart server → local WiFi only
- Zone detection → in-memory, no internet needed
- Firestore → write on zone change, graceful failure if offline

---

## Architecture

### ble_server (Dart) — writes to Firestore
Package: `dart_firebase_admin`
- On every zone change: write/update `patients/{beaconId}` document
- On every zone event: add doc to `events/` collection
- Service account JSON credentials (downloaded from Firebase Console)

### ble_dashboard (Flutter Web) — reads from Firestore
Packages: `firebase_core`, `cloud_firestore`, `firebase_auth`
- Real-time `StreamBuilder` on `patients` collection (replaces polling eventually)
- Event log from `events` collection ordered by timestamp desc

---

## Firestore Data Model

### Collection: `patients`
Document ID: beacon_id (e.g., "1", "2")
```json
{
  "beacon_id": 1,
  "current_zone": "OPD Waiting",
  "gateway_id": "GW_001",
  "rssi": -81,
  "last_seen": Timestamp,
  "zone_entry_time": Timestamp
}
```

### Collection: `events`
Document ID: auto-generated
```json
{
  "beacon_id": 1,
  "zone": "OPD Waiting",
  "gateway_id": "GW_001",
  "timestamp": Timestamp,
  "message": "Patient P-1 entered OPD WAITING"
}
```
